# Language Ontology V4

Language Ontology V4 promotes repeated language-memory wins into a persistent rule bank.

## Goal

Move some messy-language coverage from:

- model compiler
- then language memory retrieval

into:

- promoted language rewrite rules

so the same phrasing can become true zero-model coverage.

## V4 loop

1. Cold prompt: Phi-3 compiles messy language into canonical Memla actions.
2. Warm prompt: Memla reuses that translation from language memory.
3. Repeated warm success promotes clause-level rewrite rules into the language rule bank.
4. Rule prompt: Memla resolves the same phrasing through `language_rule` before language memory or the model.

## What V4 proves

- the language frontier can shrink again after memory
- repeated memory wins become explicit reusable language infrastructure
- some prompts no longer need:
  - the model
  - or language-memory retrieval

## Runtime order

1. heuristic language
2. promoted language rules
3. language memory
4. language-model compiler
5. validation
6. browser C2A execution
