# Memory Ontology V2

Memory Ontology V2 extends the governed memory lifecycle from language traces to autonomy traces.

V1 proved that language compiler wins could become governed memory objects:

- `episodic`
- `semantic`
- `rule`

V2 adds autonomy memory kinds such as:

- `autonomy_github_repo_scout`
- `autonomy_subject_followup_youtube`
- `autonomy_subject_followup_reddit`
- `autonomy_evidence_synthesis`

The goal is to let Memla remember not only phrasing translations, but successful task shapes:

- search and collect candidates
- rank against a goal
- inspect top candidates
- follow the selected subject to another source
- synthesize cross-source evidence

These traces use the same adjudication path as language memory:

- successful traces increase trust
- repeated reuse can promote from `episodic` to `semantic`
- bad reuse can decay trust or invalidate the memory

This is the bridge from "I can do one scout" to "I know this mission pattern."
